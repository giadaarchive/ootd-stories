# L's Collection of Amazing Pieces

This file documents how the wardrobe database works — what it tracks, what each property means, how to add and edit entries, and how the video archival system feeds into it.

**Notion database:** `ad079964-9690-43ae-9fa8-5a4f3ca1a9ee`

---

## What this database is

A complete inventory of every physical item in Lisa's collection — clothing, bags, shoes, jewellery, watches, scarves, eyewear. One row per item. Every item that matters has a record. If it is not in the database, it does not officially exist.

The database does four things at once:
1. **Inventory** — what exists, where it lives, what it cost
2. **Care record** — how to wash, dry, iron, and store each item correctly
3. **Wear tracking** — how often each item is worn, calculated as cost per wear
4. **Knowledge base** — why Lisa owns it, what she'd change, personal stories, research notes

## Properties

### Identity

| Property | Type | Notes |
|----------|------|-------|
| `Second best` | title | The item name. Include brand, material, colour, key descriptor |
| `Old Title` | rich_text | Previous name — kept for search/reference if renamed |
| `SKU` | rich_text | Unique identifier. Format: `BRAND-CATEGORY-MATERIAL-YY-###`. Generated by `generate_skus.py` |
| `Category` | select | `Basic` / `Basic wow` / `Wow` — how special the piece is |
| `Designer` | relation | Link to the Designer database |
| `Material Category` | relation | Link to the Material Category database |

### Financial

| Property | Type | Notes |
|----------|------|-------|
| `SGD` | number | Purchase price in Singapore dollars |
| `Retail Price (USD)` | number | Original retail price in USD if known |
| `Additional Costs` | number | Dry cleaning, repairs, alterations in SGD |
| `Total Cost (SGD)` | formula | SGD + Additional Costs |
| `USD` | formula | SGD converted to USD |
| `Savings` | formula | Retail price minus what was paid |
| `CPW (SGD)` | formula | Cost per wear in SGD — Total Cost divided by number of times worn |
| `Retail CPW (USD)` | formula | What CPW would be at retail price |

### Time

| Property | Type | Notes |
|----------|------|-------|
| `Year It's Made (first hand)` | date | Year of manufacture, not purchase. Drives the YY in the SKU |
| `Date I bought/own` | date | Date of acquisition |
| `Added` | created_time | When the row was created in Notion |
| `Age of Garment` | formula | Today minus Year It's Made |
| `Last Worn` | rollup | Most recent OOTD entry linking to this item |

### Care

| Property | Type | Valid values |
|----------|------|-------------|
| `Wash Method` | multi_select | `Spot clean`, `Dry clean`, `Machine wash`, `Handwash` |
| `Wash Temperature` | multi_select | `Cold`, `20ºC`, `30ºC`, `40ºC`, `60ºC` |
| `Drying` | select | `Line dry`, `Lay flat to dry`, `Dryer` |
| `Storage Method` | multi_select | `Cedar box`, `Drawer`, `Hanger` |
| `Ironing` | multi_select | `Do not iron`, `Steam`, `Press cloth required`, `Medium` |
| `Season` | multi_select | `Spring`, `Summer`, `Autumn`, `Winter` |

### Tags (relations to Tags DB)

| Property | Type | Notes |
|----------|------|-------|
| `Why I own it (Tags)` | relation | Pull factors — what made her buy or keep it. Uses the same tag vocabulary as the Deinfluence tracker |
| `What I'd change (Tags)` | relation | Things she'd change or doesn't love — uses the "why no" tag vocabulary |

See `DEINFLUENCE_SKILLS.md` for the full tag vocabulary and definitions.

### Status flags

| Property | Type | Notes |
|----------|------|-------|
| `Favourite` | checkbox | Items she reaches for instinctively |
| `To Wear/Style` | checkbox | Flagged to intentionally wear or style soon |
| `with/for mum` | checkbox | Gifted to or shared with mum |
| `Thanks for the memories` | checkbox | No longer in the collection — marks items for departure |

### Connections

| Property | Type | Notes |
|----------|------|-------|
| `How L Styles` | relation | Links to OOTD entries where this item was worn |
| `Fits` | formula | Count of times worn, derived from OOTD |

## Page body structure

Each entry has a page body with freeform content. The structure is still evolving — what follows describes what currently exists, not a fixed template.

**Typical sections:**

- **Item photos** — images of the piece, usually from the listing or taken by Lisa
- **Owners and Stories** — a table with `Previous Owners` (provenance, who owned it before) and `The Story` (why she bought it, what she loves, the emotional or aesthetic reasoning in her own words)
- **Research and Notes** — a table with `Research Purpose` (what she was trying to learn or test), `Hypothesis`, `Observation Log` (notes after first, fifth, tenth wear), `Verdict` (KEEP / SELL / DONATE / TBD), `Resale Plan`
- **Item detail paragraphs** — structured lines pulled from the original listing: `[Brand]`, `[Size]`, `[Actual Size]`, `[Material]`, `[Color]`, `[Specifications]`, `[Condition]`, `[Remarks]`, `[Rank]`

This structure is a work in progress. Older entries may follow different formats.

## The video archival system

### What it is

Lisa records videos about individual pieces — talking through what the item is, why she has it, what the fibre or material is like, and how to care for it properly. These videos serve as a long-form knowledge record for future reference (or future generations).

Videos are uploaded to YouTube (private or scheduled) and embedded in the Notion entry once published.

### The `Archival Recorded/shared` property

A **date** field. Set it to the date the video was recorded or publicly shared. When this date is filled in, it signals that a video exists for this item.

The video itself is embedded in the page body as a block (either a native video upload or a YouTube embed link).

### Running the archive script

Once a video is published on YouTube, run `wardrobe_archive.py` to automatically extract and fill in the entry:

```bash
python3 wardrobe_archive.py <notion_page_url> <youtube_url>
```

**What it does:**
1. Fetches the video transcript from YouTube
2. Sends the transcript to Claude with the item title as context
3. Extracts and writes back:
   - **The Story** — a prose narrative written into the Owners and Stories table
   - **Care properties** — Wash Method, Wash Temperature, Drying, Storage Method, Ironing (anything mentioned in the video)
   - **Tags** — `Why I own it` and `What I'd change` from the approved tag vocabulary
   - **Season** — if mentioned

The script only fills fields that are clearly mentioned in the transcript. It does not guess or invent values.

**Prerequisites:**
- `youtube-transcript-api` installed (`pip3 install youtube-transcript-api`)
- `.env` file with `NOTION_TOKEN` and `ANTHROPIC_API_KEY`
- `tag_id_map.json` present in the same directory

## How to add a new item

1. Create a new row in the database
2. Fill in: title, Designer (relation), Material Category (relation), `SGD`, `Year It's Made`, `Date I bought/own`
3. Leave SKU blank — run `generate_skus.py` to assign automatically
4. Add item photos to the page body
5. Write in the Owners and Stories table: who owned it before, and why she bought it
6. Fill care properties (Wash Method, Drying, etc.) — or run `wardrobe_archive.py` after recording a video
7. Set `Why I own it (Tags)` from the approved tag vocabulary in `DEINFLUENCE_SKILLS.md`

## How to read the collection

- **Most worn items:** sort by CPW ascending — lowest cost per wear = best real-world value
- **Never worn:** sort by Last Worn ascending, nulls first
- **By designer:** filter by Designer relation
- **Items to consider departing:** filter where `Thanks for the memories` is checked
- **Items needing care docs:** filter where Wash Method is empty
- **Items with video:** filter where `Archival Recorded/shared` is not empty
- **Favourites:** filter where `Favourite` is checked
